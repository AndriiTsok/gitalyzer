# Gitalyzer Documentation

This directory is the single source of truth for Gitalyzer's requirements, design, and
usage documentation.

## Layout

| Path             | Purpose                                                       |
| ---------------- | ------------------------------------------------------------- |
| `product.md`     | Product requirements (PRD) — what Gitalyzer does and why.     |
| `rfcs/`          | Numbered RFCs — technical requirements and design decisions.  |
| `rfcs/README.md` | The RFC process and index of all RFCs.                        |

## Where to start

Read [`product.md`](product.md) first — it defines the product: the problem, the two
modes, and the expected user experience.

We are currently in the **requirements phase**: the product definition is being turned
into detailed technical requirements and design decisions as RFCs. See
[`rfcs/README.md`](rfcs/README.md) for the process and the index of active documents.

Additional documentation (architecture, user guides, operational runbooks) will be added
here as the project moves from requirements into implementation.
