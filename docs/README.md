# Gitalyzer Documentation

This directory is the single source of truth for Gitalyzer's requirements, design, and
usage documentation.

## Layout

| Path          | Purpose                                                          |
| ------------- | --------------------------------------------------------------- |
| `rfcs/`       | Numbered RFCs — requirements and design decisions.              |
| `rfcs/README.md` | The RFC process and index of all RFCs.                       |

## Where to start

We are currently in the **requirements phase**. Requirements are captured as RFCs. See
[`rfcs/README.md`](rfcs/README.md) for the process and the index of active documents.

Additional documentation (architecture, user guides, operational runbooks) will be added
here as the project moves from requirements into implementation.
